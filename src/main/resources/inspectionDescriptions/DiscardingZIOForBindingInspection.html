Detects if an effect is suspiciously used in for-comprehension as a simple binding and not as a generator: 
for {
  a <- ZIO.succeed(1)
  _ = ZIO.logInfo("msg")
  b <- ZIO.succeed(1)
} yield a + b

Will highlight the expression _ = ZIO.logInfo("msg").
Fix by using generator syntax: 
for {
  a <- ZIO.succeed(1)
  _ <- ZIO.logInfo("msg")
  b <- ZIO.succeed(1)
} yield a + b
